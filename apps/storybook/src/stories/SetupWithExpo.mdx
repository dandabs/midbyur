<Title>Set up Midbyur with Expo</Title>

<Subtitle>Configuration-only reference</Subtitle>

Use @midbyur/ui with Expo Router and direct stylesheet import. No NativeWind or Tailwind app setup is required.

## Installation

Install dependencies from the repository root:

```bash
pnpm install
```

Dependencies used by the working example app include:

- @midbyur/ui
- @midbyur/theme
- react-native-css-interop
- expo-router
- expo-dev-client
- expo-font
- expo-linking
- expo-constants
- @expo/metro-runtime
- @expo/log-box
- @expo/dom-webview
- burnt

## Run Commands

Start Metro for dev-client:

```bash
pnpm --filter @midbyur/example-expo start:dev-client
```

Run native builds:

```bash
pnpm --filter @midbyur/example-expo ios:run
pnpm --filter @midbyur/example-expo android:run
```

Run web:

```bash
pnpm --filter @midbyur/example-expo web
```

## Required Configuration

### 1. Package Entry

apps/example-expo/package.json should use:

```json
{
  "main": "expo-router/entry"
}
```

### 2. Babel

Create or update apps/example-expo/babel.config.js:

```js
module.exports = function (api) {
  api.cache(true);

  return {
    presets: ["babel-preset-expo"],
  };
};
```

### 3. Expo Router Plugin

apps/example-expo/app.json should include:

```json
{
  "expo": {
    "plugins": ["expo-router"]
  }
}
```

Create route files under apps/example-expo/app, including _layout.tsx and tab pages.

### 4. Metro

Create or update apps/example-expo/metro.config.js:

```js
const { getDefaultConfig } = require("expo/metro-config");
const { withCssInterop } = require("react-native-css-interop/metro");
const path = require("node:path");

const config = getDefaultConfig(__dirname, { isCSSEnabled: true });

const appNodeModulesPath = path.resolve(__dirname, "node_modules");
const workspaceNodeModulesPath = path.resolve(__dirname, "../../node_modules");

config.resolver = config.resolver || {};
config.resolver.nodeModulesPaths = [appNodeModulesPath, workspaceNodeModulesPath];

config.watchFolders = [path.resolve(__dirname, "../..")];

module.exports = withCssInterop(config, {
  inlineRem: 15,
});
```

### 5. Shared Stylesheet Import

Import shared UI styles directly in apps/example-expo/app/_layout.tsx:

```ts
import "../../../packages/ui/src/components/styles.css";
```

If you use App.tsx instead of Expo Router:

```ts
import "../../packages/ui/src/components/styles.css";
```

### 6. TypeScript CSS Declaration

apps/example-expo/nativewind-env.d.ts:

```ts
declare module "*.css";
```

apps/example-expo/tsconfig.json should include nativewind-env.d.ts in include.

### 7. Fonts

apps/example-expo/fonts.ts:

```ts
import type { FontSource } from "expo-font";

export const MIDBYUR_NATIVE_FONTS: Record<string, FontSource> = {
  "Stack Sans Text": require("./assets/fonts/StackSansText.ttf"),
  "Stack Sans Notch": require("./assets/fonts/StackSansNotch.ttf"),
  "Stack Sans Headline": require("./assets/fonts/StackSansHeadline.ttf"),
};
```

## Troubleshooting

### Native toasts do not appear

Use dev-client flow (not Expo Go):

```bash
pnpm --filter @midbyur/example-expo start:dev-client
pnpm --filter @midbyur/example-expo ios:run
```

### Styles are missing

Check all of the following:

- apps/example-expo/app/_layout.tsx imports ../../../packages/ui/src/components/styles.css.
- Metro config wraps `withCssInterop(config, { inlineRem: 15 })`.
- Restart Metro after metro.config.js edits.

Then clear cache:

```bash
pnpm --filter @midbyur/example-expo start -- --clear
```

### Font changes not reflected

- Confirm filenames in fonts.ts exist under apps/example-expo/assets/fonts.
- Restart Metro with --clear.
- Re-run ios:run or android:run.
