    # Midbyur + Next.js Setup
  </div>

  <div className="sb-section">
    <h2>Installation</h2>

    <p>Create a new Next.js app with the App Router enabled, then add Midbyur and its dependencies:</p>

    <pre><code>{`pnpm create next-app@latest my-app
cd my-app
pnpm add @midbyur/ui @midbyur/theme next react react-dom react-native react-native-web
pnpm add -D tailwindcss @tailwindcss/postcss nativewind webpack @babel/plugin-transform-flow-strip-types babel-loader typescript @types/react @types/react-dom @types/node`}</code></pre>

    <p>Expo apps should install <code>burnt</code> with Expo's installer:</p>

    <pre><code>{`npx expo install burnt`}</code></pre>

    <p>
      If consuming <code>@midbyur/ui</code> from a workspace package, keep the Tailwind <code>@source</code> paths as shown below.
      If using a published package, adjust paths to match your distribution layout.
    </p>
  </div>

  <div className="sb-section">
    <h2>Required Configuration</h2>

    <h3>1. PostCSS</h3>

    <p>Create <code>postcss.config.mjs</code>:</p>

    <pre><code>{`const config = {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};

export default config;`}</code></pre>

    <h3>2. Tailwind CSS Entry Point</h3>

    <p>Create <code>app/globals.css</code>:</p>

    <pre><code>{`@import "tailwindcss" important;
@source "./**/*.{js,ts,jsx,tsx,mdx}";
@source "../packages/ui/src/**/*.{js,ts,jsx,tsx,mdx}";`}</code></pre>

    <p><strong>Important notes:</strong></p>
    <ul>
      <li>The <code>important</code> flag ensures Tailwind styles have high specificity and won't be overwritten after hydration.</li>
      <li>The <code>@source</code> entries must include both your app and the UI package, otherwise Tailwind won't generate the component styles.</li>
      <li>Update the second <code>@source</code> path to match your repository layout.</li>
    </ul>

    <h3>3. Next.js Configuration</h3>

    <p>Create or update <code>next.config.ts</code>:</p>

    <pre><code>{`import type { NextConfig } from "next";
import webpack from "webpack";

const nextConfig: NextConfig = {
  transpilePackages: [
    "@midbyur/ui",
    "@expo/html-elements",
    "nativewind",
    "react-native-css-interop",
    "burnt",
  ],
  webpack: (config) => {
    config.plugins ??= [];
    config.plugins.push(
      new webpack.DefinePlugin({
        __DEV__: JSON.stringify(process.env.NODE_ENV !== "production"),
      }),
    );

    config.resolve ??= {};
    config.resolve.alias ??= {};
    config.resolve.alias["react-native"] = "react-native-web";

    return config;
  },
};

export default nextConfig;`}</code></pre>

    <p><strong>Notes:</strong></p>
    <ul>
      <li><code>transpilePackages</code> ensures React Native and NativeWind dependencies are compiled for Next.js.</li>
      <li>The <code>react-native</code> alias maps to <code>react-native-web</code> for browser compatibility.</li>
      <li>Defining <code>__DEV__</code> in webpack handles client-side; server-side is handled separately.</li>
    </ul>

    <h3>4. Server Polyfill for __DEV__</h3>

    <p>Create <code>app/polyfills.ts</code>:</p>

    <pre><code>{`(globalThis as unknown as Record<string, unknown>).__DEV__ =
  process.env.NODE_ENV !== "production";`}</code></pre>

    <p>
      Import this <strong>first</strong> in <code>app/layout.tsx</code>, before any <code>@midbyur/ui</code>, 
      <code>react-native</code>, or NativeWind imports. Server Components run in raw Node.js before webpack substitution applies.
    </p>

    <h3>5. Root Layout</h3>

    <p>Create <code>app/layout.tsx</code>:</p>

    <pre><code>{`import "./polyfills";
import "./globals.css";
import { ClientProviders } from "./client-providers";

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body>
        <ClientProviders>{children}</ClientProviders>
      </body>
    </html>
  );
}`}</code></pre>

    <p>
      The <code>suppressHydrationWarning</code> is required because Miðbýur's toaster layer is client-only 
      and will mismatch during hydration if rendered on the server.
    </p>

    <h3>6. Client-Only Provider Boundary</h3>

    <p>
      <strong>Important:</strong> Do not use <code>ssr: false</code> directly in <code>layout.tsx</code> 
      because layout is a Server Component. Create a separate Client Component that wraps it.
    </p>

    <p>Create <code>app/providers.tsx</code>:</p>

    <pre><code>{`"use client";

import { MidbyurProvider } from "@midbyur/ui";

export function Providers({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return <MidbyurProvider>{children}</MidbyurProvider>;
}`}</code></pre>

    <p>Create <code>app/client-providers.tsx</code>:</p>

    <pre><code>{`"use client";

import dynamic from "next/dynamic";

const Providers = dynamic(() => import("./providers").then((mod) => mod.Providers), {
  ssr: false,
});

export function ClientProviders({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return <Providers>{children}</Providers>;
}`}</code></pre>

    <p>
      This setup prevents hydration mismatches by rendering <code>MidbyurProvider</code> only on the client, 
      avoiding the toaster layer mismatch during server-to-client hydration.
    </p>

    <h3>7. TypeScript Configuration</h3>

    <p>Make sure <code>tsconfig.json</code> includes NativeWind type definitions:</p>

    <pre><code>{`{
  "compilerOptions": {
    "types": ["nativewind/types"]
  }
}`}</code></pre>

    <h3>8. First Page</h3>

    <p>
      Pages that use interactive Miðbýur components must be Client Components. 
      Create <code>app/page.tsx</code>:
    </p>

    <pre><code>{`"use client";

import { Button } from "@midbyur/ui";

export default function Home() {
  return (
    <main>
      <Button>Hello Miðbýur</Button>
    </main>
  );
}`}</code></pre>
  </div>

  <div className="sb-section">
    <h2>Troubleshooting</h2>

    <h3>Button renders unstyled</h3>

    <p>Check all of the following:</p>
    <ul>
      <li><code>app/globals.css</code> imports Tailwind with <code>important</code>.</li>
      <li><code>app/globals.css</code> includes an <code>@source</code> entry for the UI package.</li>
      <li><code>globals.css</code> is imported by <code>app/layout.tsx</code>.</li>
    </ul>

    <h3>Styles flash and then disappear</h3>

    <p>
      This happens when Tailwind imports are missing the <code>important</code> flag. 
      The styles render initially but are overwritten after hydration. Use:
    </p>

    <pre><code>{`@import "tailwindcss" important;`}</code></pre>

    <h3>Hydration failed: server HTML did not match client</h3>

    <p>
      Do not render <code>MidbyurProvider</code> directly in the server layout. 
      Keep it behind the <code>ClientProviders</code> boundary as shown above.
    </p>

    <h3>Hero is taller than viewport on Android mobile</h3>

    <p>
      Use <code>100dvh</code> (dynamic viewport height) instead of <code>100vh</code> 
      for hero heights to account for browser chrome that compresses and expands. 
      Miðbýur's Hero component uses <code>100dvh</code> by default.
    </p>
  </div>
